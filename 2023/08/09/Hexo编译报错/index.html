昨天晚上提交了一篇文章到 GitHub 上，但是今天一早起来再想提交的时候就发现博客的 Markdown 无法通过编译，即 hexo generate 失败，告诉我哪个哪个文件找不到了。

# 官网 troubleshooting

文件明显就摆在那里，但是为什么会报错呢？于是我按照报错给的链接去到官网的 troubleshooting 看看有无解决方案：

官网提示说有可能是我的文件名、分类或者标签的名字混淆了大写和小写，确实有这种可能，昨天晚上提交的那篇文章的一个 tag 是 English，而之前我类似文章都是定义成 english 的，提交上 GitHub 之后可能在 public 文件夹下存在 english 和或 English 文件夹，可能因为 Git 的 bug 或是啥的，然后出现出现了不可名状的错误。

可能官方给出来的确实是这种问题的解决方法之一，不过对我不适用，pass！

# Stack Overflow

# Juliet

先摆个原贴链接 npm - How to resolve Node.js: "Error: ENOENT: no such file or directory" - Stack Overflow，要是我指定的这种方法仍然行不通的话，就去试试其它的办法吧！

这个问题是六年前有人提出来的，既然搜索引擎将老古董置顶，那肯定说明这里面的方法是有效的。

在 Threads 里，有个高赞回复：

Your app is expecting to find a file at  /home/embah/node/nodeapp/config/config.json  but that file does not exist (which is what  ENOENT  means). So you either need to create the expected directory structure or else configure your application such that it looks in the correct directory for  config.json .

他认为要么是你文件找不到了，要么就是项目的配置信息出现了错误。原来如此！即使是题主的 Node.js web application 还是我的 hexo 项目，都是拥有一个配置文件和 node_modules（后面会说），因为它们都基于 nodeJS。

我找到了个 YAML Validator 工具，用来检测 hexo 项目中的主题和根目录下的_config.yml 语法是否错误。工具链接在这里，网站名叫做 code beautify，还是一个挺多实用小工具的网站，可以收藏一下。将两个文件夹下的配置信息 paste 到网站后，显示 VALID YAML，说明配置文件并没有出错。

虽然这个方法对我并没有奏效，但是还是将它贴在这里，因为可能你们的解决方法就是恰好是这个。毕竟我还是相信还是有很多大头虾们不会注意 yml 文件里需要冒号后面接空格 space😝😝😝

After going through so many links and threads and getting frustrated over and over again, I went to the basics and boom! it helped. I simply did:

为了找到解决方法，我也算是 plagued by this error for a long time，键入 npm install 后，没想到我不仅本问题没有 fix，反倒引出了新的问题，具体输出就是一堆的 npm ERR 。

针对这个问题，我找了一下，发现很多时候安装 npm install 的时候并不能直接将当前的 node_modules 直接一键安装到本地。

为了解决这个问题，可以下载安装新的 node 安装包，或者如果本地有多个新老项目为了方便的管理各个 node 的版本问题，使用 nvm, 这是一个很好用的 node 版本协调控制工具。权衡利弊，我选择了比较高级的后者。

nvm 的常用命令如下:

  • nvm install stable ## 安装最新稳定版 node
  • nvm install <version> ## 安装指定版本
  • nvm uninstall <version> ## 删除已安装的指定版本
  • nvm use <version> ## 切换使用指定的版本 node
  • nvm ls ## 列出所有安装的版本
  • nvm ls-remote ## 列出所有远程服务器的版本
  • nvm current ## 显示当前的版本
  • nvm alias <name> <version> ## 给不同的版本号添加别名
  • nvm unalias <name> ## 删除已定义的别名
  • nvm reinstall-packages <version> ## 在当前版本 node 环境下，重新 全局安装指定版本号的 npm 包
  • nvm alias default [node 版本号] ## 设置默认版本

想要查看具体的 nvm 安装步骤，可以查看这里，下载链接在这里

  1. 装好 nvm 之后，便可以在命令行键入 nvm 查看信息，无需配置环境。
  2. 随后执行 npm install -g npm 命令，更新 npm。
  3. 删除 node_module 文件，因为后面执行的 npm install 会生成新的 node_module
  4. 删除 package-lock.json 文件，因为它会限制你执行 npm install 所生成的 node_module 信息（后期会自行生成）
  5. 执行 npm install ，succeed！

到此 npm install 问题解决。尝试执行 hexo generate ，发现也没有报错了！看来这个方法还是很适合我的，一举两得。

估计就是 node_module 在什么不可外力的作用下出了点错误，导致 Markdown 编译不过关，重装 node_module 就 OK 了。

其实和我的解决方法类似，只不过多了项步骤 npm cache clean --force ，无论如何，能够解决问题就好囖，深究下去只会越陷越深。Reference Here - Stack Overflow
