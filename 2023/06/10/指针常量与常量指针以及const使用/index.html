# 1. 指针常量与常量指针的区别

# 1.1 指针常量

常量指针和指针常量的区别_飞翔的荷兰橙的博客 - CSDN 博客指针常量就是一个指针修饰的常量，格式如下：

int* const p; //指针常量

指针常量的地址不能被修改，但是它指向的内容可以修改

# 2. const 的使用

C++ 里 const int* 与 int const* 有什么区别？ 面对 c++ 或者 c 中纷繁的 const 和 *，需要遵循的金科玉律就是 const 默认作用于其左边的东西，否则作用于其右边的东西，换种说法就是 const 在 * 的左边，对象值不能修改。const 在 * 的右边，地址不能修改。

  1. int* const
    1. 指针常量
  2. const int*
    1. 常量指针
  3. const * int
    1. 常量指针
  4. const int* const
    1. 指向常量的指针常量
  5. int const * const
    1. 指向常量的指针常量
  6. int const * const *
    1. 指向常量的指针常量的指针

常见的估计就这上面六种了吧，复杂的估计我也遇不到！

# 3. 引用 &

  1. 要知道引用和指针都是地址，但是引用只是指针的一个别名而已，要是引用指向的值改变，引用的值也会跟着改变。
  2. 引用在使用时必须被初始化，而指针不是必须的。
  3. 引用初始化后不能改变所指向的对象，但是指针可以。
  4. 指针可以有多级但引用只能有一级。有 int ** , 但是 int && 并不代表引用的引用，而代表右值引用。
  5. 当指针和引用作为函数参数的时候，指针传递参数会生成一个临时变量，引用传递的参数不会生成一个临时变量。

实操了一下，终于想起引用的用法了。无非就是指针的别名，实质上还是指针。常见于函数参数传递上。以下参考 C++ 函数传参 int a,int *a,int &a,const int &a 的区别_鱼日天的博客 - CSDN 博客

TypeDefinitione.g.
int a值传递void Swap(int a, int b)
int * a地址传递void Swap(int *a, int *b)
int & a引用传递void Swap(int &a, int &b)
const int &a常量引用传递void Swap(const int& a,const int &b)
int const &a常量引用传递void Swap(int const &a,int const &b)

地址传递就不用说了，可以修改地址，自然也就能够修改原参数的值，注意在调用的时候要写成 Swap(&a,&b) 这种形式。

值传递和引用传递最大的区别就是它们是否可以更改原参数的值，值传递传入的显然就是形参（一个数字），不能改变原参数的值。而有引用传递传入的参数实际上就是原参数，可以改变原参数的值。

引用传递和常量引用传递的区别就是常量引用传递无法修改原参数的值，但常量引用传递和值传递仍不同。值传递传入的是一个数字，常量引用传递传入的仍然是原参数，只不过不允许修改罢了。
