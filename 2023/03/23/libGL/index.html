Linux环境下Qt报错libGL

发表于 2023-03-23
  • Yume Shoka
首页 Study

today i first execute the Qt software under the enviroment of Ubuntu, but somethings was occurred : -1: error: cannot find -lGL: No such file or directory.When I first saw it, I think the criminal leads to this problem is me...

But actually it's not my problem until i change my coding way. The bug is still over here??? I have no choice but to figure out what cause it.

i try to paste this bug on the browser and find the answer. The solution is as follow...

因为 Qt 找不到 OpenGL 的动态链接库（libGL.so）Qt 默认在 /usr/lib/ 目录下查找动态链接库，但是很多 Linux 发行版将 OpenGL 链接库放在其它目录，例如我使用的是 CentOS 7，OpenGL 链接库位于 /usr/lib64/ 目录，而对于 Ubuntu，OpenGL 链接库位于 /usr/lib/i386-linux-gnu/mesa/ 目录。只要我们把 libGL.so 拷贝到 /usr/lib/ 目录，或者在 /usr/lib/ 目录下为 libGL.so 创建一个链接，就能解决问题。显然第二种办法更好。Refer to

# 1. download Locate Lib

sudo apt-get install mlocate
sudo updatedb

so you can execute the locate order in the Ubuntu Command Line later...

# 2. download revelant Lib

sudo apt-get install mesa-common-dev
sudo apt-get install libgl1-mesa-dev libglu1-mesa-dev

# 3. final step?

maybe this step is dispensable, i.e. not important. i guess this order just play a role in LOCATE? BUT I do it anyway.so you can try whether you can solve this probiem with just two steps above and without this step?

locate libGL.so
