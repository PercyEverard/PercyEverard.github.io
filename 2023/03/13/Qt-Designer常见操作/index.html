# 1. 如何在 UI 界面上使用自己创建的类呢？

自己创建的类一般都是要继承某一个组件才能够实现的。在设计 UI 界面的时候，先把这个父组件拖出来，然后右键选择提升。如果你之前没有对组件进行提升过的话，那么你就要自己手动创建这么一个提升的对象，也就是你自己创建的控件。注意控件名称要和你命名的一模一样，不然的话可能无法识别出来。

# 3. Qt->QAction->QAction::triggered

我们经常可以看到 toggle ()、triggered ()、clicked () 这三个函数在槽函数的信号位置出现，既然它们是作为信号存在的，那么它们之间又有什么呢？

其实这三个信号都是按钮点击后发射的信号，区别在于：

  • clicked () 用于 Button 发射的信号
  • triggered () 用于 QAction 发射的信号，原型：void triggered (bool checked = false);
  • toggle () 用于 ChekBox, 非开即关，原型：void toggled (bool);
action_systemTray_love = new QAction(QIcon(":/image/image/image/gary_love.png"), u8"我喜欢");
connect(action_systemTray_love, &QAction::triggered, this, &MainWidget::on_Lovebtn_clicked);

我对 triggered 的用法感到疑惑，于是上网查了一下，发现它的确是可以作为信号发生器存在在 connect 函数中的。不过要注意的是后面的槽函数形式，&MainWidget::on_Lovebtn_clicked。别以为里面有个 clicked 就是触发动作自然而然地联想到信号发生器，其实它是在 ui 界面上 “转到槽” 操作自动生成的对点击响应的函数，一式两份，一份在你的 mainWidget 中，另外一份在头文件里。并且一定得实现什么逻辑功能才行，如果不实现的话，Qt 在执行程序的时候会报错。题主之前不小心给控件生成了一个点击槽函数，因为本来就不需要实现点击功能也就撂那没管了，没成想执行的时候因为它报错了...
