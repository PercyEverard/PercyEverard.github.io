# 2. 友元函数和友元类

友元函数

在定义一个类的时候，可以把一些函数（包括全局函数和其他类的成员函数）声明为 “友元”，这样那些函数就成为该类的友元函数，在友元函数内部就可以访问该类对象的私有成员了。

友元类

一个类 A 可以将另一个类 B 声明为自己的友元，类 B 的所有成员函数就都可以访问类 A 对象的私有成员。

# 3. inline 内联函数

在系统下，栈空间是有限的，假如频繁大量的使用就会造成因栈空间不足而导致程序出错的问题，如，函数的死循环递归调用的最终结果就是导致栈内存空间枯竭。

避免了频繁调用函数对栈内存重复开辟所带来的消耗。

inline 的使用时有所限制的，inline 只适合函数体内部代码简单的函数使用，不能包含复杂的结构控制语句例如 while、switch，并且不能内联函数本身不能是直接递归函数（即，自己内部还调用自己的函数）。

建议：inline 函数的定义放在头文件中

# 4. auto 隐式推导

这个是类型推导关键词，C++11 引入了 auto 和 decltype 关键字实现类型推导，通过这两个关键字不仅能方便地获取复杂的类型，而且还能简化书写，提高编码效率。我们可以通过 auto 隐式地指定数据的类型，那我们又不给他指定数据类型，那谁来做这件事情呢？答案是编译器！隐式类型定义的类型推导发生在编译期，编译器能够自动推断出这个变量的类型，而不需要显式指定类型。

说了这么多，auto 好像可有可无？的确是这样的，它的出现只不过是帮助我们省下了对已知类型变量的类型定义代码的编写工作，如果你勤奋点的话也可以不用 auto。在 C++11 之前，定义了一个 stl 容器需要给迭代器写比较长的定义，但是有了 auto 的话就简单多了！auto 的推导其实和 template 函数模板参数的自动推导挺像的，不过 auto 是不能用作函数模板的参数推导的。

#include <iostream>
#include <map>
#include <vector>
using namespace std;

int main(){
    vector<int> resultMap;
    for(auto it = resultMap.begin(); it != resultMap.end(); ++it)
    {
	cout << "hello!" << endl;
    }
    return 0;
}

# 6. C++ 迭代器（STL 迭代器）iterator 详解

要访问顺序容器和关联容器中的元素，需要通过 “迭代器（iterator）” 进行。迭代器是一个变量，相当于容器和操纵容器的算法之间的中介。迭代器可以指向容器中的某个元素，通过迭代器就可以读写它指向的元素。从这一点上看，迭代器和指针类似。详情可以参考这里，这里讲解了迭代器的辅助函数还有功能分类。

迭代器按照定义方式分成以下四种。

  1. 正向迭代器，容器类名::iterator 迭代器名；
  2. 常量正向迭代器，容器类名::const_iterator 迭代器名；
  3. 反向迭代器，容器类名::reverse_iterator 迭代器名；
  4. 常量反向迭代器，容器类名::const_reverse_iterator 迭代器名；

每种容器都定义了一对名为 begin 和 end 的函数，用于返回迭代器。容器不为空时，begin 返回的是容器的第一个元素。要注意的是 end 返回的迭代器指向的是 vector 的末端的下一个，通常叫他是超出末端迭代器（off-the-end iterator），它指向一个不存在的元素，如果 vector 为空，则 begin 和 end 返回的迭代器相同。

# vector 迭代器的自增和解引用运算

  1. 自增操作 ++：表示向前移动迭代器，指向下一个元素。可以结合 int 的自增理解：对于 int 类型来说，表示 “加 1”，而对容器来说，是 “下移一个位置”。如果当前迭代器指向第一个元素，则 ++iter 表示指向第二个元素。

  2. 解引用操作 *：可以用来访问迭代器所指向的元素。

  • 发现 iteration 有时候可以被 size_type 和 size_t 所代替，只需要理解 size_type 是容器概念，用于表示容器长度，是 STL 定义的。而 size_t 不是容器概念，它的引入只是增强了平台的可移植性。通俗一点说就是用 size_type 你总是没错的！

下面使用 cin 用来标准输入读取数据，常用的输入方法有 cin>>、cin.get ()，cin.getline ()

#include <iostream>
#include <vector>

using std::cin;
using std::cout;
using std::endl;
using std::vector;

int main(){
	vector<int> arr;
	/*
	for(vector<int>::size_t i = 0; i != 10; ++i)
	{
		//cin >> temp;
		arr.push_back(i);
	}
	*/
	int temp;
	for(int i = 0; i < 5; ++i)
	{
		cin >> temp;
		arr.push_back(temp);
	}
	for(int j = 0; j < arr.size(); ++j)
	{
		cout << arr[j] << "\t";
	}
	cout << "normal read" << endl;
	for(vector<int>::iterator it = arr.begin(); it != arr.end(); ++it)
	{
		*it *= 2;
		cout << *it << "\t";
	}
	cout << "read with iterator" << endl;
	for(vector<int>::const_iterator cit = arr.begin(); cit != arr.end(); ++cit)
	{
		// *cit *= 2; const_iterator read only!
		cout << *cit << "\t";
	}
	cout << "we can't change this container" << endl;
	for(vector<int>::reverse_iterator rit = arr.rbegin(); rit != arr.rend(); rit++)
	{
		cout << *rit << "\t";
	}
	cout << "read the container reverse!" << endl;
	//vector<int>::iterator middle = arr.begin() + arr.size()/2;
	//cout << middle << "get middle data" << endl;
	return 0;
}

# 7. exit 函数

#include <iostream>
#include <cstdlib>
using namespace std;

void print();

int main(){
	print();
	return 0;
}

void print(){
	cout << "you can see this \n" << endl;
	exit(EXIT_SUCCESS);
	cout << "you can't see this \n" << endl;
}

如果 main 函数在执行过程中遇到 return 的话，那么这段程序就会停止执行。遇到其它函数倒也不会出现这种直接退出的情况，只有一个函数例外，那就是 exit。

想要调用 exit 函数，还得提前引入头文件 <cstdlib>。

只要主函数的逻辑遇到 exit，就像遇到 return 一样，结束程序。这里的 exit 退出还分有两种情况，一种是成功退出，另外一种是程序因为故障中断，这里就要看 exit 后面的返回的整形变量啦！我们常见的 exit（0）就是成功返回，还有另外一种也是表达成功返回的方式就是 exit（EXIT_SUCCESS）。如果你见到代码 exit（1）、exit（-1）或者 exit（EXIT_FAILURE），那就说明是因为故障退出程序。

上面的 EXIT_SUCCESS 和 EXIT_FAILURE 是 C++ 的命名常量，其实还有一点奇怪的就是，主程序都结束了，你的 exit 函数返回值给谁啊？其实这个实参可以理解成是由程序返回到计算机操作系统的退出代码，以指示程序退出是成功结束还是故障的结果。

# 8. int、int&、int * 之间的关系

# int

int 是 32 位有符号整型，范围是 - 2,147,483,648 到 2,147,483,647。将变量名作为实参，它的传递是单向的。

# int &

int & 是一个变量的引用，可以理解成是一个非 void 变量的别名。引用不是独立的变量，变量和变量引用都共享同一内存单元，系统并不会给引用另外分配存储单元。（设指针要另辟内存）

int a = 1;
int & b = a;

如果你尝试输出 a 和 b 的地址，你会发现它们两的地址都是相同的，因为它们两本来就是在同一块存储单元下的。引用既然被初始化了（指已经被声明引用某个变量了），就不能再被声明说引用了其它变量。也就是说一个引用自打出生以来就跟着一个变量，不离不弃。

引用还可以被引用，形成俄罗斯套娃...(不知道有啥用)

注意，引用前面的数据类型指的是你要引用变量的数据类型，记住他两要一样就行了。但是 & 有时候也可以用来取地址符啊，我咋知道它到底是指取变量的地址还是指引用呢？这就得看有没有数据类型了，如果有数据类型的话就说明我这里是引用的意思，否则就是取地址的意思。

int & a = b; different from & a = b;

为什么要有引用这个东西呢？我们都清楚如果用变量名作为实参，其传递方向是单向的，而用引用作为实参其传递方向是双向的。这个时候就有人说，指针也可以做到啊！其实引用能做的事情指针都能做，但是指针麻烦啊，所以就出现了这么个引用。于是乎在这个时候，变量和变量引用任何一个发生变化，另外一个都会发生相应的变化！

int * 的话没啥好讲的，创建指针数据类型嘛。这里提提 int & 使用的注意事项。

  • 将变量的引用的地址赋给一个指针，此时指针指向的是原来的变量
  • 可以用 const 对引用加以限定，不允许直接改变该引用的值，但是可以改变原变量的值去改变引用的值
  • 可以用常量或表达式对引用进行初始化，但此时必须用 const 作声明。

比如说：

int i=5;
const int &a=i+1;
	\|||/
	 \|/
	  |
int i = 5;
int temp=i+1;
const int &a=temp;

加上一个 const，能够保证引用值不发生变化，还有一个作用就是允许了生成临时变量这种操作... 所以我们经常能够看到 const 和 & 走在一起！
