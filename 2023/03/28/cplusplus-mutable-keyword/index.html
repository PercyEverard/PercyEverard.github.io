cplusplus 中的 mutable keyword

发表于 2023-03-28
  • Percy Feng
首页 编程拓展

在 C++ 中 mutable 关键字是为了突破 const 关键字的限制，被 mutable 关键字修饰的成员变量永远处于可变的状态，即使是在被 const 修饰的成员函数中 --> 原文链接 /summary：对于 const 修饰的变量就没必要再用 mutable 修饰了，这个 mutable 针对的情况是 const 修饰的函数无法访问到该类的变量... 就很烦

在 C++ 中被 const 修饰的成员函数无法修改类的成员变量，成员变量在该函数中处于只读状态。然而，在某些场合我们还是需要在 const 成员函数中修改成员变量的值，被修改的成员变量与类本身并无多大关系，也许你会说，去掉函数的 const 关键字就行了。可问题是，我只想修改某个变量的值，其他变量希望仍然被 const 关键字保护。

现在有个场景，我们想获取函数被调用的次数，代码如下：

class Animals
{
private:
    int number;
    mutable int count;
public:
    Animals();
    ~Animals() = default;

    void getNumber() const;
    int getCount() const;
};

这里我们想要获取 getNumber 函数被调用次数，普遍的做法是在 getNumber 函数里对成员变量 count 进行加 1 处理，可是 getValue 被关键字 const 修饰啊，无法修改 count 的值。这个时候 mutable 派上用场了！我们用 mutable 关键字修饰 count，完整代码如下：

#include <iostream>
#include <cstdio>
using namespace std;
class Animals
{
private:
    int number;
    mutable int count;
public:
    Animals();
    ~Animals() = default;

    void getNumber() const;
    int getCount() const;
};

Animals::Animals() : number(1),count(0){}

void Animals::getNumber() const
{
    count++;
}

int Animals::getCount() const
{
    return count;
}
int main()
{
    Animals a;
    for(int i = 0; i <= 5; i++)
    {
        a.getNumber();
    }
    cout << a.getCount() << endl;
    return 0;
}

被 mutable 修饰的成员变量 count 在 getNumber 函数里进行加 1 计数，编译运行输出为 6，这样既保护了其他成员变量，又能达到我们单独修改成员变量 count 值的目的。
