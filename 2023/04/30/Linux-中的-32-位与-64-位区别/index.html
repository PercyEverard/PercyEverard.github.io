Linux中32位与64位区别

发表于 2023-04-30
  • Yume Shoka
首页 知识大陆

在通用 PC 领域，不论是 windows 还是 linux 界，我们都会经常听到 "32 位" 与 "64 位" 的说法，类似的还有 "x86" 与 "x86_64"，"i386" 与 "amd64"，这两组概念之间有着怎样的联系和区别呢？想要讲清楚这一点，我们可以从三个层面展开，分别是硬件、系统、软件，因为他们三者都可以是 64 位或者说是 32 位的。

# 硬件

早期的 Intel 处理器，比如说 i386,486，586，或者说奔腾系列的 CPU 架构，都是 32 位的，这里的 32 位指的是寄存器是 32 位，虚拟地址是 32 位。这一代被称作是 X86 架构。

后来 AMD 公司抢占了先机，率先开发出 64 位的 CPU 架构，并且能够完全兼容 X86。并且得到了广泛的推广，它们推出的 CPU 架构统称为 amd64 架构，支持 amd64 指令集。

当然，Intel 也不甘示弱，马上开发出了支持 amd64 指令集的，并且能够兼容 x86 架构的 CPU 架构。它在非 Linux 系统中被称为是 x64 架构，在 Linux 中叫法比较特殊，被称作是 x86_64 架构。不过都是 x86 架构的 64 位拓展。reference

# 系统

只有硬件升级了，才有软件迭代的机会。在 64 位 CPU 架构出现以前，操作系统都是 32 位。只有 64 位 CPU 架构出现以后，才慢慢地涌现出 64 位的操作系统。这里以 x86_64 架构为例，x86_64 完全兼容 x86 架构，所以运行在 x86 架构上的 32 位操作系统，也同样能够运行在 x86_64 架构上，只不过不能发挥出 64 位 CPU 架构的全部硬件功能而已。而 64 位操作系统就是为了最大限度的发挥 64 位 CPU 架构的硬件能力而开发的，但是它无法在 32 位的 x86 上运行。

另外，不论是 windows 还是 linux，不同于应用程序，驱动程序都需要加载到内核空间中，因此它的类型 (32 位或 64 位）需要跟操作系统完全相同。reference

# 软件

在系统上运行的应用软件同样有 32 位和 64 位之分，老的应用软件运行在 32 位 CPU 及 32 位系统中，属于 32 位软件。出现了 64 位系统之后，也出现了专门运行在 64 位系统，能够最大限度使用 64 位系统的扩展功能的原生 64 位程序。

对于 C 程序来说，64 位程序相比 32 位程序最大的区别就是 int，long 以及指针数据类型的长度都是 64 位的，可以使用 64 位的虚拟地址空间。另外在 64 位系统中，为了兼容数量众多的老的应用程序，系统都会增加一个虚拟层，用来翻译 x86 指令，这样 32 位程序也可以在 64 位系统中运行 (windows 和 linux 都支持)。同样在 64 位系统中，使用 64 位编绎器一般也可以选择编译成 32 位程序 (gcc 使用 - m32 编译成 32 位可执行文件)。

应用程序能否运行，关键取决于操作系统，而不是 CPU 架构。64 位的应用程序无法在 32 位的操作系统上执行，可见无论是操作系统和 CPU 架构还是应用程序和操作系统，都是向上兼容的。reference
