HEXO-THEME Redefine 更新

发表于 2023-04-23
  • Yume Shoka
首页 Blog

今天正常在 Git 提交文章的时候，发现 Git 竟然报错了。 hexo c && hexo g && hexo d 指令已经烂熟于心了，肯定不是这里出了问题。我查看 Git 报错信息，Print 如下：

Github: https://github.com/EvanNotFound/hexo-theme-redefine
current version is v1.2.0,latest version is v2.1.2
=====================================================
Redefine v1.2.0 is outdated, please update to v2.1.2!

原来是 Redefine 主题需要更新了。我循着报错给的链接找到了 Refine version2.1.2 详细信息，并且根据提示更新主题。

Github 上写着有两种方法用于更新，npm 和 git，但我发现它们两个的效果并不是一样的。最后把我文件夹的层级关系搞得一团糟，导致我不得不重新删除博客底下的 node_module 和 theme 文件夹并且重新操作。

$ cd your-hexo-site
$ npm install hexo-theme-redefine@latest

$ cd your-hexo-site
$ git clone https://github.com/EvanNotFound/hexo-theme-redefine.git themes/redefine

两条指令都是在博客目录下进行的。npm 指令用于生成 node_module 文件，文件内包含各种组件。可见这次更新带来了不少的新功能，比如包含的 nodejieba 组件，用来进行中文分词。甭管它哪里用到，加进去就完事🐶（如果不下载组件的话，会导致 Git 提交信息时报错）！而 git 指令则是用来下载主题，如果博客底下含有 theme 文件夹，则会在 theme 文件夹下生成这次的主题文件 Redefine，否则它就会先创建一个 theme 文件夹，随后再在 theme 下创建 Redefine 文件夹。

更新完组件和主题后，需要在 hexo 配置文件_config.yml 下修改 theme 栏信息，将原来的 hexo-theme-redefine 改成 redefine。

还没结束！你打开网站发现主题显示的页面信息和原来不一样了，这是因为更新主题同样会更改主题下的配置文件。存放在配置文件中的个人偏好和个人信息都需要自己重新手动输入，这也是比较麻烦的一点。

至此更新完成！
