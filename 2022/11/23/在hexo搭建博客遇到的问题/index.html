之前我用的主题感觉索然无味，于是我尝试换了一个主题，结果发现了新大陆！激动的话我就不在这里详细张开来讲了，我会放在另外一篇文章里，这里主要讲讲我在 Hexo 上搭建 Next 遇到的问题以及解决方法。

# Hexo 搭建了博客显示不出 Categories 和 Tags

# 在主题配置文件中取消注释 Categories 和 Tags

博客文件中一共有两类配置文件，一类是 Hexo 的配置文件、另一类是主题配置文件。首先要做的事是将主题配置文件打开，找到 Menu Settings 并且把 Categories 和 Tags 取消注释。

# 新建页面

在你的 source 文件夹中新建文件

hexo new page categories
hexo new page tags
hexo new page about
hexo new page schedule

# 修改 index.md

上面除了 categories 和 tags 我还写入了关于和日程表页面，只要你有需求就可以在主题配置文件中将他们打开。它们的处理方式都是类似的，所以我将它们放在一起讨论。

打开 source 文件夹下的 index.md 文件后，开头只有 title 和 data，我们要做就是在后面加上一句 type: categories ，其它也是同理。注意冒号后面有空格。

之前有人说用 layout: categories 的方式也可以，我尝试了一下，发现已经失效了。原因可能是因为当前的 hexo 已经不支持 layout 这种格式了。

# hexo 建立同级 categories

之前我对 categories 还没有分级和同级的概念，直到我对一篇文章想要运用多个 categories 的时候才初现端倪。网上的人都说 hexo 已经不支持同级分类了（但是我确实实现同级分类了）

如果我们在 outline 如何命名：

categories: 
- ML
- Homework

或者说是这样

categories: [ML, Homework]

上面两种方法效果一样，Homework 都是 ML 下的子类，你要是有 categories: [Homework] 这个目录，你会发现它们并不是相同的！因为一个是 Homework 一个是 ML 下的 Homework。

要是真想 ML 和 Homework 有同样的级别，可以尝试一下以下方法：

categories:
- [ML]
- [Homework]

# hexo 主页不显示全文

在 outline 下面，你需要加入一段简介，作为这篇文章的介绍，随后再加入 <!-- more --> ，那么下面的内容便可以被隐藏。若是不加简介的话， <!-- more --> 隐藏下面的功能便会失效。

# hexo 常用命令

hexo n "name"       # 新建文章
hexo new page "name"  # 新建页面
hexo g                # 生成页面
hexo d                # 部署
hexo g -d             # 生成页面并部署
hexo s                # 本地预览
hexo clean            # 清除缓存和已生成的静态文件
hexo help             # 帮助

但是我通常都是使用 hexo clean && hexo g && hexo d , 因为其他的很少用到！
